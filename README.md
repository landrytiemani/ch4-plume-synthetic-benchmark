**A Controlled Synthetic Sentinel-2 L1C Benchmark for Methane Plume Segmentation: U-Net Variants and a Physics-Gated Dual-Branch Architecture**

[@sonamog](https://github.com/sonamog) · [@landrytiemani](https://github.com/landrytiemani) · Faith Edafetanure-Ibeh

Paper: `paper/main.tex` | Code: https://github.com/landrytiemani/ch4-plume-synthetic-benchmark

## Overview

This repo is fully self-contained. Starting from zero, it downloads methane plume data from public APIs,
builds a curated event catalog with spatial TRAIN/VAL/TEST splits, acquires matching Sentinel-2 L1C
chip pairs via Google Earth Engine, and trains five segmentation architectures on physics-injected
synthetic plumes. No external project dependencies or pre-existing data files are required.

Pipeline steps:

0. **Build plume catalog** — download Carbon Mapper (public API, no auth) + NASA EMIT (free account), build binary masks, match to S2 scenes, assign spatial TRAIN/VAL/TEST splits
1. **Acquire S2 chips** — export Sentinel-2 L1C event/reference chip pairs via GEE, download via rclone
2. **Build training manifest** — align plume masks to chip grids, curate quality flags
3. **Train benchmark models** — physics-informed synthetic plume injection on five architectures
4. **Evaluate** — held-out synthetic TEST metrics, inference speed, publication figures

The benchmark does **not** claim real-label methane detection accuracy. All plume masks and methane
absorption signals are generated by code; the real Sentinel-2 chips provide only the background texture.

## Step 0 — Build the Plume Catalog

This step downloads plume data from two public sources, converts rasters to binary masks,
finds matching Sentinel-2 L1C scene pairs for each plume event, and assigns deterministic
TRAIN/VAL/TEST splits by spatial block (0.25° grid, 70/15/15, seed `ch4syn-v1`):

```bash
GEE_PROJECT=your-gcp-project bash scripts/build_plume_catalog.sh
```

Sub-steps (each can be skipped individually — see `SKIP_*` variables in the script):

| Sub-step | What it does | Output |
|---|---|---|
| 0a | Carbon Mapper catalog (public API, no auth) | `data/carbon_mapper/carbon_mapper_ch4_permian_*.csv` |
| 0b | Carbon Mapper plume TIF download | `data/carbon_mapper/plume_tifs/` |
| 0c | NASA EMIT CH4 download (free account) | `data/emit/products/` |
| 0d | Unified label catalog | `data/labels/plume_label_catalog.csv` |
| 0e | Binary mask conversion | `data/labels/source_masks/` |
| 0f | GEE S2 L1C scene matching | `data/raw/sentinel2_l1c/sentinel2_l1c_reference_match_catalog.csv` |
| 0g | Spatial-block split assignment | `data/raw/splits/segmentation_split_catalog.csv` |

Skip individual sub-steps:
```bash
SKIP_CM=1 SKIP_EMIT=1 GEE_PROJECT=your-gcp-project bash scripts/build_plume_catalog.sh
```

## Step 1 — Acquire Sentinel-2 L1C Chips

```bash
GEE_PROJECT=your-gcp-project \
DRIVE_FOLDER=CH4_Plume_L1C_S2_pairs \
bash scripts/acquire_s2_data.sh
```

This does four things:
1. Queries GEE for the best low-cloud Sentinel-2 L1C event + reference scene for each plume event
2. Queues 512×512 px chip exports (bands B2, B3, B4, B8, B11, B12 × event + reference = 12 bands) to Google Drive
3. Waits for you to confirm all GEE tasks are `COMPLETED` at https://code.earthengine.google.com/tasks
4. Downloads chips from Drive to `data/raw/sentinel2_l1c/exports/` via rclone
5. Audits downloaded files

**Smoke test** (20 rows, ~2 min):
```bash
LIMIT=20 GEE_PROJECT=your-gcp-project bash scripts/acquire_s2_data.sh
```

**Skip steps you've already done:**
```bash
SKIP_MATCH=1 SKIP_EXPORT=1 bash scripts/acquire_s2_data.sh   # re-download only
SKIP_MATCH=1 SKIP_EXPORT=1 SKIP_PULL=1 bash scripts/acquire_s2_data.sh  # audit only
```

## Step 2 — Build Training Manifest

Once chips are downloaded, align the plume masks to each chip's pixel grid and curate quality flags:

```bash
bash scripts/build_training_manifest.sh
```

Outputs:
- `data/training_l1c/segmentation_training_manifest.csv` — one row per chip with pair status and mask stats
- `data/training_l1c/segmentation_training_curated.csv` — adds `usable_for_training`, `sample_weight`, `label_quality`

---

## Step 3 — Run the Benchmark

Train all five models on synthetic plumes injected into the real L1C backgrounds:

```bash
bash scripts/run_synthetic_publication_benchmark.sh
```

Default settings: 40 epochs, batch 8, patch 128×128, 8192 patches/epoch, 1024 VAL patches, 2048 TEST patches.

**Smoke test** (1 epoch, 2 models):
```bash
EPOCHS=1 PATCHES_PER_EPOCH=64 VALIDATION_PATCHES=32 TEST_PATCHES=64 \
MODELS="unet phys_tau_net" \
bash scripts/run_synthetic_publication_benchmark.sh
```

## Step 4 — Inference Speed

After training:
```bash
python scripts/benchmark_inference_speed.py \
  --benchmark-dir data/models/synthetic_publication_benchmark \
  --manifest data/training_l1c/segmentation_training_curated.csv \
  --split TEST --patch-size 128 --batch-size 8 \
  --warmup-batches 10 --timed-batches 50 --device auto
```

## Method Scope

**Included in this benchmark:**
- Real Sentinel-2 L1C event/reference background chips (publicly available from Copernicus)
- Plume label catalog built from Carbon Mapper (public API) and NASA EMIT (free account)
- TRAIN/VAL/TEST split assigned by spatial block (0.25° grid, greedy 70/15/15, fully reproducible)
- Methane-sensitive SWIR event/reference feature stack (B11/B12 log-ratio, Varon-like ratio, SWIR residual)
- Beer-Lambert B12 attenuation from synthetic optical-depth plume fields
- Irregular synthetic plume morphology (random walks, dilation)

**Explicitly excluded:**
- Carbon Mapper or EMIT real-label training claims
- Operational real-world plume detection performance
- Cross-sensor generalization claims
